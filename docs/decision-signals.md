# Decision Signals

Decision signals summarize the state that drives buy, hold, sell, risk, and market-light guidance. Inputs may include price trend, volume, technical indicators, chip structure, fundamentals, news, announcements, sentiment, portfolio context, and market phase.

Signals must explain important sources and limitations. Missing or unsupported data should be identified instead of silently treated as neutral. Intraday or estimated data should be marked as estimated, stale, or partial when relevant.

When signal logic changes, test the affected report path and include before/after examples for user-visible wording or structure changes.
