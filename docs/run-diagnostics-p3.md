# Run Diagnostics P3

P3 makes diagnostics visible in user-facing surfaces. Web/API should expose concise status for configuration, providers, notifications, and runtime health. Messages should explain what users can do next. Existing analysis and notification flows must continue if diagnostics degrade.

For Web changes, run `npm run lint` and `npm run build` under `apps/dsa-web`. For backend/API changes, run the closest Python checks.
